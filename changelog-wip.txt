BotC v1.5.1 - Feb 13, 2026
Changes and features:
- Util functions are better named and utilized now.
- Init sequence is now automatically run on world load.
- World protection is now handled in the init sequence.
- Trouble Brewing is now loaded by default.
- World icon is now the same as the pack icon.
- Consolidated icons in the grim slightly.

- Voice chats are not auto-initialized.
- Storyteller options (currently just clock time) should be disabled if you are not a Storyteller.
- Storyteller prefix in tab is not handled by init sequence.
- Spectator prefix in tab is not handled by init sequence.
- Spectators not teleporting to the correct location on the new map on join and game reset.
- Spectators are shown as "Not in a house" to Storytellers.
- joined_game scoreboard objective isn't initialized.
- Vanilla UI elements (like the chat) should display on top of custom ones.
- Add Server screen doesn't have a custom background.
- Night Order screen only displays Trouble Brewing.
- New seats in the grim display the wrong characters.
- New seats in the grim display the wrong names.
- "Teleport Players to Seats" button is not updated for the new map.
- The player 7 character icon is displaying in the grim when it should not.
- "Light Pyre" teleports players to the old location.
- Voting doesn't work for new seats.
- Clock arm targeting wrong entity type.
- Adventure mode is not forced on first join.
- Players can sometimes spawn in the wrong place on first join.
- Players who left while in a game and returned after a reset are not re-initialized.
- Players leaving before the game does not reduce player count for the purposes of character distribution displays.
- New seats' "Dead" state is not reset with game reset.
- Some seats around town use trapdoors instead of regular doors for their backs.
- The Magenta and Lavender players are swapped in some places.
- Skulls can appear on player seats due to the team suffixes not getting reset correctly.
- Block sounds are enabled during night and disabled during the day, instead of the inverse.
- Server configs should just be included in the regular config folder.
